## 1. Базовый URL API (`VITE_API_BASE_URL`)

**Вопрос:** Какое значение `baseURL` у `apiClient` по умолчанию?

**Варианты:**

- [ ] A) Пустая строка — пути вида `/api/session/check` (same-origin через Traefik)
- [ ] B) `/api` — пути вида `/session/check`
- [ ] C) Полный URL из env (`https://xxyyzz.localhost/api`)
- [ ] D) Свой вариант

**Пояснение:**  
При схеме «фронт и API на одном домене» достаточно относительных путей. `envDir` в Vite указывает на корень проекта — переменные можно держать в общем `.env.template`.

**Рекомендация:** A или B (единообразие путей в коде).

**Ответ:** `/api` — пути вида `/session/check`.

## 2. Автообновление сессии по таймеру

**Вопрос:** Нужно ли автоматически вызывать `check` / `refresh` до истечения `accessExpiresAt`?

**Варианты:**

- [ ] A) Нет — только ручной `check()` / `refresh()` и вызов при монтировании `SessionCheck`
- [ ] B) `setTimeout` за N секунд до `accessExpiresAt` → `check()`
- [ ] C) `setInterval` с фиксированным интервалом (указать: \_\_\_)
- [ ] D) Свой вариант

**Пояснение:**  
Бэкенд ротирует access на `/session/check` при истечении. Фронт может проактивно дергать check до expiry, чтобы не ловить 401 на других запросах.

**Рекомендация:** B с запасом ~1 мин до `accessExpiresAt`.

**Ответ:** да, нужно. С запасом в 90 до `accessExpiresAt`.
